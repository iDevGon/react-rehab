# 미션 01: 필터 가능한 목록

공통 요구사항: [common-requirements.md](./common-requirements.md)

## 목적

controlled input, 파생 목록, 조건부 렌더링, empty state를 연습합니다. 현재
검색어와 카테고리 필터에 따라 보이는 인벤토리 목록을 계산하되, 원본 데이터는
변경하지 않는 것이 목표입니다.

## 대상 파일

`apps/web/src/missions/filterable-list/Exercise.tsx`

## 구현할 내용

여행 인벤토리 목록에 검색과 카테고리 필터를 구현합니다.

스타터는 세 개의 인벤토리 아이템, 검색 input, 카테고리 select를 렌더링합니다.
처음에는 컨트롤을 조작해도 목록이 바뀌지 않습니다. 현재 컨트롤 값을 state로
관리하고, 그 값으로 보이는 아이템을 계산하며, 매칭 결과가 없을 때 empty state를
렌더링하세요.

## 요구사항

- `Search items`에 입력하면 아이템 이름 기준으로 필터링됩니다.
- 검색은 대소문자를 구분하지 않습니다.
- 카테고리를 선택하면 해당 카테고리 아이템만 보입니다.
- `All categories`는 모든 카테고리를 보여줍니다.
- 검색과 카테고리 필터는 함께 동작해야 합니다.
- 매칭되는 아이템이 없으면 `No items match` 문구가 포함된 empty state를 보여줍니다.
- 원본 인벤토리 배열을 직접 변경하지 않습니다.

## 완료 조건

- 미션별 테스트 명령이 통과합니다.
- 검색어와 맞지 않는 아이템은 숨겨집니다.
- 카테고리 선택 시 다른 카테고리 아이템은 숨겨집니다.
- 검색과 카테고리를 조합해도 올바른 결과가 나옵니다.
- 파생 결과 목록이 비어 있을 때만 empty state가 나타납니다.

## 수동 확인 절차

1. `pnpm dev`로 앱을 실행합니다.
2. Filterable List 미션을 엽니다.
3. 검색 필드에 `mug`를 입력합니다.
4. `Trail Mug`만 보이는지 확인합니다.
5. 검색어를 지우고 `Clothing` 카테고리를 선택합니다.
6. `Atlas Jacket`만 보이는지 확인합니다.
7. 아무것도 매칭되지 않는 문자열을 입력합니다.
8. empty state가 나타나는지 확인합니다.

## 테스트 명령

```bash
pnpm --filter @react-rehab/web test src/missions/filterable-list/Exercise.test.tsx
```

## 테스트가 검증하는 동작

- 검색이 이름 기준으로 결과를 좁힙니다.
- 카테고리 선택이 카테고리 기준으로 결과를 좁힙니다.
- 매칭 결과가 없으면 empty state를 렌더링합니다.

## 제약사항

- 정답 결과를 하드코딩하지 않습니다.
- 아이템을 CSS로만 숨기고 결과 목록에는 남겨두는 방식은 피합니다.
- 이 미션에는 커스텀 훅이 필요하지 않습니다.

## AI 에이전트 평가 기준

AI 리뷰어는 controlled input 사용, state에서 보이는 아이템을 파생하는지, 원본
데이터를 유지하는지, 행동 테스트 통과 여부, 의미 있는 empty state 렌더링을
확인해야 합니다.
