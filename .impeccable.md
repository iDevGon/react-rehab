## Design Context

### Users

React Rehab is used by developers who want to rebuild direct hands-on React
coding fluency after relying heavily on AI agents. They use it locally while
working through focused missions, checking requirements, running tests, and
reviewing their own implementation attempts.

### Brand Personality

Precise, restorative, disciplined. The interface should feel like a serious
developer workbench: focused, technical, and calm without becoming cold or
generic.

### Aesthetic Direction

Use a precision workbench aesthetic. The UI should feel closer to a high-end
IDE inspection surface, lab bench, or technical training console than a
marketing page or playful course app. Prefer a dark navigation rail, light
working surfaces, blueprint-like structure, clear hierarchy, and deliberate
accent color. Avoid generic SaaS cards, bright AI gradients, glassmorphism, and
decorative noise.

### Design Principles

- Make the current mission and next coding action obvious at a glance.
- Use strong layout structure and restrained color instead of decoration.
- Treat requirements, verification, commands, and preview as distinct work
  zones.
- Keep typography compact, readable, and purposeful for repeated practice.
- Preserve accessibility, responsive behavior, and keyboard-friendly controls.
